# Schema Design Justification

## Schema Structure Justification

The metadata schema deliberately separates analysis-level metadata from dataset-specific metadata to ensure scalability and consistency across multiple files. Analysis-level sections (dataset_provenance, normalization, event_selection, systematic_summary, usage_guidelines) capture information that applies uniformly to the entire analysis, while the datasets array contains file-specific details that vary between individual HDF5 files. This separation prevents redundant repetition, ensures consistency when metadata is updated, and scales naturally to analyses with many datasets.

Each section addresses a specific reproducibility requirement. The dataset_provenance section documents data origin and generation procedures, enabling users to understand generator differences and assess systematic uncertainties. The normalization section provides information required to convert event weights to physical cross-sections. The event_selection section defines the analysis phase space, allowing users to determine kinematic coverage. The systematic_summary section catalogs uncertainty sources for proper error propagation. The usage_guidelines section prevents common misuse patterns by documenting recommended practices and known limitations. This structure transforms implicit analysis conventions into explicit, machine-readable metadata that supports long-term dataset preservation and reuse by physicists who were not involved in the original analysis.

## Inclusion and Exclusion Decisions

Every field addresses a specific reproducibility or reuse risk identified through systematic inspection of the HDF5 files. Fields like weight_definition clarify the relationship between weight_mc and weights_nominal, which is undocumented in the files themselves. The systematic_sources_present list maps the 100 ensemble weights to their physical meaning. The compatible_datasets field prevents incorrect dataset combinations that could introduce double-counting or normalization errors. Each inclusion decision directly mitigates a concrete failure mode in dataset reuse.

The decision to explicitly mark unknown values as "unknown" rather than omitting fields or fabricating plausible values preserves scientific integrity. Omitting fields suggests the information is unnecessary, potentially leading users to proceed without critical metadata. Fabricating values would be scientifically dishonest and could propagate incorrect assumptions through subsequent analyses. Explicit "unknown" markers make metadata gaps immediately visible, forcing users to seek authoritative information rather than proceeding with potentially incorrect assumptions. This approach prioritizes correctness over convenience and transforms the schema into a self-documenting checklist of required information for reproducible analysis.

## Expected User Interaction

A physicist encountering these datasets without prior knowledge would use the metadata schema to understand dataset roles, select appropriate weights, and avoid common pitfalls. The dataset_provenance section immediately clarifies generator differences (multifold_sherpa.h5 uses Sherpa while others are unknown), alerting users to potential systematic variations. The purpose field distinguishes primary analysis datasets from validation and background samples, preventing misuse such as treating background as signal.

When constructing weighted histograms, users consult usage_guidelines to identify weights_nominal as the recommended weight column, then check normalization to discover that integrated luminosity is unknown and absolute cross-section measurements require external information. The systematic_summary section informs users that ensemble weights encode systematic uncertainties and bootstrap weights provide statistical uncertainties, enabling proper uncertainty propagation within the limitations of available metadata. For dataset combination, users verify compatibility in compatible_datasets, then review limitations to understand that relative normalization factors must be obtained from analysis documentation.

The "unknown" markers function as safety warnings that prevent silent propagation of errors into physics results. When users encounter integrated_luminosity: unknown, they understand that luminosity normalization requires external information and cannot proceed without it. This design philosophy ensures that metadata gaps do not silently compromise scientific validity, supporting reproducible research and safe dataset reuse across research groups and time scales.
